Areeb Majeed


Feature Learning Research Design Coding Testing
MVF 1: Password and 2FA Support 4 4 5 6 2
Justification: This feature requires us to build support for passwords as well as 2FA codes. It would require study behind how much passwords can be stored and 2FA codes generated. From the design perspective, we are supposed to make a page where we can show the passwords and 2FA for each website the user stores.
MVF 2: Cloud Storage 4 2 1 6 1
Justification: The cloud backup feature requires study on how to store passwords and 2FA codes safely in a database. In our case, it would require us to build a database schema and connect it with our web application. From designing perspective, there isn't a lot of to do in this case, other than showing a page which shows the number of passwords stored in the database, etc. The testing of this feature would include checking whether the database is storing the user credentials correctly or not.
MVF 3: Master Key Encryption 3 2 N/A 2 1
Justification: This feature includes the storage of passwords and 2FA keys in the database, but in an encrypted form so that user details stay safe and sound. Naturally, study of encryption algorithms is required. Since this is a pure backend feature, there is little to no design involved. The testing of the feature would simply involve checking whether the passwords are stored in encrypted form in the database or not.
MVF 4: Web Application 16 8 6 8 3
Justification: This can be considered as one of the largest feature of the project as it involves creating a web-application where the users can interact, access, modify or delete their passwords and 2FA keys. In addition to that, the users should also be able to login, register an account at the website. Naturally, this feature would heavy research, design and coding. In addition to that, it would involve testing features such as user management, etc.
MVF 5: Export Passwords and 2FA Seeds 3 1 1 2 1
Justification: This would involve research on what data serializing format to use (for ex: JSON, XML or CSV). From design perspective, this would creating a page where the user can press a button to export the passwords. The code running behind this feature should compile all passwords and 2FA seeds, and then put them in a JSON file and prompt the user to download it. Testing would simply check whether the passwords are exported correctly or not.
EVF 1: Find Leaked Passwords 4 1 N/A 3 1
Justification: To build this feature, it is essential to extract database dumps from the Internet and then parse them into database which can be queried to check whether the user stored passwords are leaked somewhere on the Internet or not. Naturally, a lot of research would be required to prepare the list of these database dumps. From design perspective, there's nothing involved. We only need to use the existing "Notifications" feature to send a notification to the user.
EVF 2: Organize Passwords 2 1 2 3 1
Justification: There's little to no research required for this feature, but in order to build this feature, there's extensive knowledge required in backend programming (PHP). From design perspective, we need to build an additional filter where the users can sort their passwords into tags and categories.
EVF 3: Password Change Notifications 1 1 2 2 1
Justification: We need to design a page to allow the user to set remainders when the password needs to be changed for critical websites such as banks and wallets. From coding perspective, we need to build a script that runs once every day and checks for user set remainders and if there's any password change due, it sends a notification to the user. Testing this would involve checking whether the user receives a notification on the set date.
