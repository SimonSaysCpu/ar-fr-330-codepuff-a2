Resources and Tools

Name GitHub
Description GitHub is used to platform used to host code (using Git versioning system). In addition to that, it also enables developers to collaborate on a project.
Rationale We're using GitHub to host our code files as well as collaborate on the project without overwriting each other's work.
Specific Version N/A
Cost Free (Student Plan)
Alternatives BitBucket, GitLab
Name Amazon Web Services (AWS)
Description Amazon Web Services is a cloud hosting provider from e-commerce giant Amazon. It provides services such as dedicated, virtual services in addition to a plethora of other APIs and products.
Rationale We are using AWS to host this report and project.
Specific Version N/A
Cost Free (Student Plan)
Alternatives Google Cloud, DigitalOcean, IBM Cloud
Name PHPStorm
Description PHPStorm is an IDE developed specifically for PHP. It supports error prevention, code analysis and code refactoring. In addition to that, it also supports PHP frameworks such as Laravel, Symphony, CakePHP, etc.
Rationale As we are coding our web-application in PHP, we will be using PHPStorm to write the project code.
Specific Version 2018.3.6
Cost Free (Student Plan)
Alternatives VSCode, Sublime Text, Brackets
Name Laravel
Description Laravel is a web-framework for PHP built to make web-development faster and easier due to the built-in features such as Authentication, Email Sending, etc.
Rationale It would be too time consuming to rewrite basic features such as Authentication, User Management and would be considered reworking the wheel, therefore we'll be using Laravel to simplify the development process for us.
Specific Version 7.21.0
Cost Free (Opensource)
Alternatives Symfony, CakePHP, CodeIgniter
Name Nginx
Description Nginx is a web-server known for its stability and high performance. It can also be used as a reverse proxy or load balancer.
Rationale Together with FastCGI, we'll be using Nginx to serve the content to the users (after execution using PHP).
Specific Version 1.18.0
Cost Free (Opensource)
Alternatives Apache HTTP Server, LiteSpeed
Name MySQL
Description MySQL is an opensource RDBMS software based on the SQL language. It's mostly used together with PHP.
Rationale We'll be using MySQL to store the user information and encrypted passwords in our MySQL database.
Specific Version 8.0.21
Cost Free (Opensource)
Alternatives MongoDB, SQL Server 2019

Contract

Expectations:

The expectations for the team CodePuff during the duration of Building I.T Systems (COSC2625) should all be agreed upon, as it is our duty to meet these requirements. With these expectations, they will also be following all RMIT guidelines for Academic Integrity purposes. Any and all acts against these guidelines, revolving the contract and RMIT's Academic Integrity will be dealt with appropriately.


Within this project assignment, there will be guidelines in place within this contract that the team CodePuff must follow. These guidelines are but not limited to:

  • All meetings to be attended; Not attending these meetings can directly impact your grade for the course and alongside all your teammates grade.
  • All meetings to be recorded with the consent of everyone within the meeting. No meeting will commence if there is a lack on consent from every team member present. This will also return a direct impact on grades.
  • Meetings that do happen, to happen minimum of once a week, with more required when necessary. This to ensure all team members are on the same page of the project alongside to keep everyone up to date professionally.
  • Activity of messages should and will be consistent. All messages that are sent within group communications should be checked regularly to make sure any team members with questions or help are not left in the dark regarding any project work.
  • All attendee's follow all guidelines put inplace by RMIT and the course. Failure to do so can directly impact grades for this course, and potentially other courses.

Section regarding Specific Tasks, Responsibilies and due dates:

Regarding the assignment for COSC2625, the project will have specific details that will have to be followed and kept in mind as the progression to the final product becomes apparent.

Project specific requirements this contract has agreed on are, but not limited to:

  • With every official due dute of the assignment for the course, finishing two days early as it can give us a grace period to reassure all features work and the clients are satisfied.
  • Goals within the project and team are kept clear and concise. Personal goals and achievements must take into factor other team members goals.
  • Roles that are assigned are assigned for a reason, therefore following what deadlines that have been placed regarding the area you are assigned to. All roles must work within their area, however are happy to help others if need be.

The Process of Dealing with Unmet Expectations:

If potentially, there are requirements that are not met and/or problems that have arised, necessary actions will be required to deal with the risk that could impact everyone.

If there is team discourse, the issue will be addressed in an appropriate manner to resolve the issue. Any bullying of any means will not be tolerated and any arguments will be dealt with appropriately and in a timely fashion.

If there are issues regarding the morality of where some information was gathered from, necessary actions will be required from the team, as in the removal of information or editing of information. If not done so, the information will be picked up by the education staff and will directly affect the team as a whole.

An Agreed Method for Feedback:

During the project, all team members will receive feedback on how they are progressing through consistent check ups on their progress within team messages and also how often they contribute to the Github and other project necessary applications. With this, if problems arise during the process of the project, receiving feedback through peers will be necessary via messages. As we hope no problems arise, each individual will still receive feedback to make sure everyone is on the same page.

Expected Frequency of Responses:

As mentioned, there will be group messages to keep contact with each other on how everything is going in relation to the project. However, frequency of responses may vary as we strive for consistency everywhere within our project.

For every message that is a question, it is expected for every person to answer if necessary to the message. If not necessary, the required people should respond if they can in a timely matter, no delaying if possible. Ideally we would be striving for same day response within a reasonable time. The same goes for when another member needs assistance in the project, to have aid in their matter as soon as possible.

Tools used within a weekly flow:

During the duration of the project, tools should be used consistently and frequently to meet deadline requirements. All tools being used will include but not limited to Github, Amazon Web Services, PHPStorm, Laravel, Nginx and MySQL. The use of these applications are expected to be used often, ideally weekly, so that all deadlines required by the course are met and some goes for any personal deadlines the team have set.
