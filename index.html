Project Description

Our project Passfree, is a web-based password manager for websites and applications that users can utilize to provide them with safety and to prevent password loss or leaks while the services that we offer are in effect. Notably, Passfree’s main features will be support for passwords as well as 2FA seeds which are often used as a second layer of protection by businesses such as Google, banks and cryptocurrency wallets. This then enables the user to have an area in the cloud system for which they can save their passwords and save them time for memorising and typing their passwords.

In addition to that, the users would be able to access the passwords anytime they wish using the web-application available. Should you want access to your passwords, we allow exportation of your passwords when asked anytime with the two factor authenticating keys that you use for them. To top it off, our services also utilizes a master key encryption to ensure that your passwords are secure and cannot be easily accessed by anyone. Moreover, our extra features will not only notify you if your passwords have been compromised so you can change them, but it will also send you timely reminders to update them if you want to ensure maximum security. And for those who have many accounts, we allow you to organise your passwords so you can separate your most important accounts from the least important ones.

User Personas

Desirable Outcomes

Main Features:


Password and 2FA Support

Description: The web-application will support passwords and 2FA (TOTP). Users would be able to store their passwords and 2FA keys securely with the password manager.

Validation: A successful implementation would mean that the users are able to copy the stored passwords and generated OTPs from the application.

User Story: As an old grandfather, I want to be able to copy and paste my passwords I already have into what I am using, so that I do not have to remember the passwords for websites I use.


Cloud Backup

Description: The passwords and 2FA seeds created by the user would be stored on the cloud. In order to prevent any leaks or data theft, the user information would be encrypted by a master key.

Validation: In order to test this feature, the user would have to initially store some passwords on the application and then delete it from their device. After they delete the application, they must reinstall it and log back in to see if they are able to access their passwords again without having to add them again.

User Story: As a teacher, I want to be able to cloud save all my students passwords securely with no hassle. I want to do this because it is readily available for when my students need them but also do not have a physical holding of these passwords.


Master Key Encryption

Description: A secure password manager should be able to safeguard the user passwords, even in the event of a hack, leak, or theft. Therefore, the passwords stored on the cloud would be encrypted by a master key provided by the user. This ensures that even if the data is leaked or hacked, the hackers would not be able to access the passwords.

Validation: Each time the user installs the application on a new device, he/she would have to re-enter their master key to decrypt the existing passwords.

User Story: As a senior level manager, I want to be able to encrypt all of my passwords and stored them on the cloud at the same time, so that when I access my sensitive information, it can not be accessed by anyone else but myself with the encryption key.


Web Application

Description: The product will come with a web-application. Users would be able to use this application to generate new passwords and 2FA codes, store their existing passwords, and export information.

Validation: The user should be able to access their passwords, 2FA codes, generate and store new passwords, and export the passwords with the web-application.

User Story: As a University Student, I want to be able to have access to all my passwords, store new ones and export them all in the same place. I want to do this because I work on my laptop for both personal and professional uses.


Export Passwords and 2FA Seeds

Description: With the application, the users would be able to export their passwords and 2FA keys in case they decide to discontinue their use of the application.

Validation: To test this feature, the user should be able to click on the Export button and then be able to download a JSON file containing all their passwords.

User Story: As a Security Analyst, I want to export all of my passwords and 2FA keys, so that when I am leaving work to work from home, I have all my passwords necessary for the websites needed to be used.



Extended Features:


Organize Passwords

Description: The users would be able to organize their passwords into different categories such as Entertainment, Games, Work, etc.

Validation: In order to validate this feature, the user should be able to create different categories and then tag a password to a particular category.


Find Leaked Passwords

Description: Often websites store the user passwords in plaintext and whenever they are hacked, the passwords end up on the dark web. This feature would involve detecting and finding the leaked passwords and informing the user to update their password automatically.

Validation: To check the successful implementation of this feature, the user should store a generic password such as "p@ssword" and the application should advise them to change it as soon as possible as the password is already leaked on the Internet.


Password Update Remainder

Description: Websites such as banks and wallets often require the users to update their password every few weeks. The user should be able to set reminders to update their passwords frequently on sensitive websites such as banks.

Validation: A successful implementation would mean that the user is able to set a reminder for a website such as a bank to update their password every 3 weeks. The user should see an alert at the application as well as get an email when they are due for a password change.

Project Risk 1: Lack of skill

Not all members are highly proficient in Javascript and PHP we will need to self-study and learn those skills out of our own volition to be able to complete the project ourselves without having to jeopardize our project ideas and force ourselves to improvise into something new in the last minute. We also intend for the team members lacking knowledge to seek help from other members that may be more experienced to ensure that the lack of skill will not prevent us from progressing.

Project Risk 2: Lack of originality

Our idea is not completely new and may lack originality to it. However, we hope to remedy that by being able to improve on those that are currently existing with both a better UI/UX design and functionality. We have also come up with features for our password manager that will set us apart from competitors such as the 2FA and reminder feature.

Project Risk 3: Scheduling and timing

Even if we give ourselves a succinct timeline to follow, there are many factors and variables that can affect our ability to reach those set deadlines which would cause a pushback of any other activities we have planned in our timeline and that would cause stress and last minute rush to have this completed. In order to minimise this risk, we aim to plan ahead and alert other members if there are any last minute obstacles that may hinder us working on the project so that a suitable solution can be found.

As we continue in the semester we all have other subjects that may take more of our time. We may lose our balance and in worst case, neglect the work we have to do as a group. Team Risk 3: Personal commitments

For most of us it will be a work commitment which changes team meeting schedules to be able to suit them as we require all memeber to be active in team meetings. Otherwise, they will not be able to have an equal say to our project. As a group, we will need to organise meeting times and coordinate around everyone's schedule to ensure that we have time for both project related and personal commitments.
